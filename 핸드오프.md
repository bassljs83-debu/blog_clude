# 핸드오프 노트 — 블로그 자동 포스팅 프로젝트
_이 파일 하나만 읽으면 어느 컴퓨터에서든 이어서 작업할 수 있습니다._
_최종 갱신: 2026-07-08 (맥북에서 git 전환)_

> ⚠️ **연속성은 이제 git(비공개 GitHub)으로 관리한다.** 구글 드라이브는 폐기.
> 다른 컴퓨터에서 이어가려면 `git clone` 후 이 파일부터 읽으면 됨.
> `.env`는 git에서 제외(각 컴퓨터 로컬 보관).

## 이어서 시작하는 법
1. `git clone <저장소>` 후 이 폴더를 Claude Code 작업 폴더로 지정
2. Claude에게 **"핸드오프.md 읽고 이어서 하자"**
3. 아래 "다음 할 일"부터 진행
4. `.env`는 로컬에 별도 준비 (양식은 Phase 1에서 생성 예정)

| 단계 | 내용 | 호스트 |

### 콘텐츠 전략 — ⭐ 2트랙 분리 (프로젝트 생존의 핵심)
2026 구글: **진짜 경험 +22% / AI 짜깁기 -71%**, 대량 AI생성은 스팸정책(scaled content abuse) 위반 → 사이트째 제재. 그래서:
- **정보/비교/구매가이드형** (대량생산 주력) → 경험 주장 안 함. 스펙·가격·리뷰종합·비교표 기반. "이런 기준으로 골랐다"
- **경험 후기형** (소수 정예) → 실제 구매 상품에만. 진짜 사진·진짜 디테일. 클러스터의 신뢰 앵커

## SEO 원칙 (2026 구글 로직 연구 결과)
1. **Experience가 최강 신호** — 우리 방향이 맞음. 단 AI 티 나면 -71% 표적 → 문체검증이 SEO 방어선
2. **AI Overviews가 리뷰/제휴 클릭 28~45% 잠식** → 빅키워드 X, **롱테일+경험+인용가능 구조**로
3. **Helpful Content는 사이트 단위** → 블로그 전체 people-first. 소개/개인정보/문의 페이지 필수. 토픽 클러스터
4. **구조화·인텐트 일치**: 롱테일 제목 · 두괄식 TL;DR · 질문형 H2(PAA) · 스펙 표 · 하단 FAQ
5. 애드센스/기술: 글당 800자+, 이미지 1~2장, Core Web Vitals

---

## 생성기 필수 골격 (템플릿 강제 슬롯)
1. 롱테일 제목 `[상황]+[대상]+[상품]+[후기/방법]`
2. 두괄식 3줄 요약 (featured snippet + AI 인용)
3. 질문형 H2 (People Also Ask 대응, 각 H2 첫 문장은 직답)
4. 스펙 비교 표 (기계 추출용)
5. 하단 FAQ
6. 경험/대가성 문구 (대가성 문구 필수)

### 검증 루프 (차별점)
- **팩트체크**: 4종 적발 — 과장(수치)·단정(정석/무조건)·검증불가(조건의존)·안전누락(부작용/한계)
- **AI 문체 탐지**: "~할 수 있습니다" 반복·균일 문단·접속사 남발·리스트 남발 교정. **단점은 리스트 말고 문장에 녹인다**
